Usability testing round 7 (opens in a new tab)

Usability testing round 6 (opens in a new tab)
  • This round was to look at the declarations for each party within the flow, to ensure they were in the correct place and that the wording made sense to the users.
  • The guide for usability testing (opens in a new tab) is kept in sharepoint.
  • The prototype was created and run in Figma with information pre-completed to make it easy for the user to click through and not be distracted by what data to complete.
  • Four flows were created, relating to four tasks, one for each stage requiring a declaration/sign off.
  • Flow 1: (opens in a new tab) For the producer to finish off the initial information, and signing the declaration.
  • Flow 2: (opens in a new tab) For the carrier to confirm they have picked up the waste.
  • Flow 3: (opens in a new tab) To confirm the waste has been picked up.
  • Flow 4: (opens in a new tab) For the receiver to confirm the waste has been received.

  • There is a 'Start a new waste movement' link to sign in as a carrier. They can then step through and add all the information, then confirm their confirmation. This is all that is needed for a carrier. (NOTE: I have hacked the code for this version so that no refresh is required.)

All items can then be reviewed prior to clicking to confirm their confirmation. (NOTE: I have hacked the code for this version so that no refresh is required.)
