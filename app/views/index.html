Version 17 (April 2023)
Created for round 9 of usability testing.
  • This round of testing is to test the flow for non-hazardous waste.
  • Added a new page to request any special handling requirements (this is not compulsory).
  • Added a new page to check if there are any POPs in the waste. This area will need more work and is serving as a placeholder so we know that POPs need to be included.
Version 16 (April 2023)
Created for round 8 of usability testing.
  • This round of testing is to carry out some A/B testing on adding the hazardous component details.
  • Version 16a - There is a single page for adding all the component details.
  • Version 16b - There is a single page for adding the component name, concentration and related hazard codes, but the user must select to add another.
Version 15 (January 2023)
Created for round 5 of usability testing.
  • This round of research was to check whether previous small changes made as a result of previous research worked for the users. The changes are documented on our design mural board .
  • A scenario document was put together to help create more realistic flows for the users.
  • The code for this prototype is a duplicate of v14.3 and then some hacking and hardcoding to make it as realistic for the user as possible.
Version 14 (Thursday 22 September 2022 - today)
Created for round 4 of usability testing with carriers.
  • There is a 'Start a new waste movement' link to sign in as a carrier. They can then step through and add all the information, then confirm their confirmation. This is all that is needed for a carrier. (NOTE: I have hacked the code for this version so that no refresh is required.)
  • However, if you wish to show as a producer continuing; if you go home (via the breadcrumbs) and then 'Continue a waste movement' the tasklist for the producer will be shown. All items can then be reviewed prior to clicking to confirm their confirmation. (NOTE: I have hacked the code for this version so that no refresh is required.)
  • If required, there is a scenario that was put together for the showcase a while ago. I have added extra information for the producer and receiver details as well. Depends on whether the usability testing is based on a specific scenario to see if they can complete everything or letting them free!
